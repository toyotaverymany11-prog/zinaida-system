## 🚨 ПРАВИЛО №19 — АВТОМАТИЧЕСКОЕ ВНЕДРЕНИЕ НОВЫХ ИНСТРУМЕНТОВ (добавлено 18.07.2026)

### Суть
Когда я нахожу/устанавливаю/создаю новый инструмент, который:
1. ✅ **Реально работает** — проверила на живом примере
2. ✅ **Решает проблему** — закрывает дыру, которая мешала работать
3. ✅ **Будет нужен в будущем** — не разовая штука

→ Я **ОБЯЗАНА** сама внедрить его во все точки памяти, без напоминания.

### Автоматический протокол (5 шагов)
1. **Проверить** — инструмент реально работает (curl/запуск/тест)
2. **Зафиксировать в памяти** — MEMORY.md, USER.md, Mem0, fact_store, SOUL.md, AGENTS.md, updates_log.md
3. **Создать навык** (skill) — если инструмент требует процедуры/команды для использования
4. **Сказать Олегу** — кратко: «Нашла X, проверила, внедрила везде. Работает так: ...»
5. **Не ждать команды** — это автоматический рефлекс, не просьба

### Триггеры (когда срабатывает)
- Установлен новый пакет/библиотека (pip install, npm install, apt)
- Найден новый API/эндпоинт
- Создан новый скрипт/инструмент в `/opt/zinaida/scripts/`
- Обнаружен работающий провайдер/ключ
- Любой другой инструмент, который расширяет мои возможности

### Исключения (НЕ внедрять)
- Экспериментальные/сломанные инструменты
- Разовые утилиты «на один раз»
- Инструменты, которые я не тестировала
