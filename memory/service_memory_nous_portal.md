## 4. ПОЧЕМУ $42 УЛЕТЕЛИ ЗА МИНУТУ — АНАЛИЗ ПЕРЕРАСХОДА

### Причина 1: Астрономическая цена модели

Одна простая операция (100 токенов входа + 100 токенов выхода):
- Вход: 100 × $0.03 = $3.00
- Выход: 100 × $0.18 = $18.00
- **Итого: $21.00 за одну элементарную операцию**

### Причина 2: Архитектура Hermes Agent (subagent-driven)

Hermes Agent не делает 1 вызов на задачу. Он разбивает каждую задачу на подзадачи, каждую выполняет через 3 вызова:
1. **Исполнитель** — выполняет задачу
2. **Проверяющий спецификацию** — проверяет что результат соответствует плану
3. **Проверяющий качество** — проверяет стиль, безопасность, ошибки

**1 логическая задача = 3+ LLM-вызова.** Если задача состоит из 2 подзадач — 6 вызовов.

### Причина 3: Скрытые расходы

| Что | Стоимость |
|-----|-----------|
| **Tool calls (вызовы инструментов)** | $2.50 / 1000 вызовов |
| **Web search** | $10.00 / 1000 + стоимость токенов запроса |
| **Контейнеры (1GB RAM, 20 мин)** | $0.03 / сессия |
| **Файловое хранилище (сверх 1GB)** | $0.10 / GB / день |
| **Cache Write** | $6.25 / 1M токенов (плата за операцию записи) |
| **Cache Read** | ~10% от стоимости входа (~$0.50/1M) |

**ПОДЛЯНКА:** Внутренняя аналитика Hermes Agent НЕ учитывает побочные вызовы, повторные попытки и записи в кэш. Реальные расходы выше показанных на 20-50%.

### Сценарий перерасхода Олега:

1. Запустил агента с GPT-5.6-sol-pro
2. Агент начал загружать документы (12.89M токенов за 98 запросов)
3. Каждая операция ~$21 (100 вход + 100 выход токенов)
4. После 2 операций: $42
5. Баланс $30 → исчерпан → кредит $10 → истрачен → минус $10
6. Задача НЕ завершена — агент оборвался на середине

**ИТОГ:** $42 за одну незаконченную задачу. 98 запросов, 12.89M входных токенов, 85K выходных.

## 5. ПРОБЛЕМЫ СОВМЕСТИМОСТИ

Модель `openai/gpt-5.6-sol-pro` может быть несовместима с текущей версией Hermes Agent (v0.16.0). Причины:
- Модель слишком новая — агент использует каталог моделей с сервера, который обновляется раз в час
- Модель могла не пройти полную проверку безопасности
- Ограничения правительства США на доступ к новейшим моделям

При ошибке несовместимости Hermes НЕ останавливается — переходит на другой вариант модели, но продолжает сжигать деньги.

## 6. СТРАТЕГИИ ОПТИМИЗАЦИИ

### Стратегия 1: Мозговой штурм vs Реализация (рекомендуемая)

**Фаза 1 — Планирование (дешёвая модель):**
- DeepSeek V4 Pro ($0.44/$0.87) или Claude Sonnet 4.5 ($2/$10)
- Генерация идей, план, структура, алгоритм
- Расход: копейки

**Фаза 2 — Исполнение (дорогая модель, если нужно):**
- GPT-5.6-sol-pro или Claude Opus 4
- Чётко сформулированная задача, минимум токенов
- Расход: контролируемый

### Стратегия 2: Управление балансом

- $20 Plus = для повседневных задач с дешёвыми моделями
- Для дорогих моделей — пополнять вручную под конкретную задачу
- Устанавливать мысленный бюджет: "на эту задачу готов потратить $X"
- Если надо работать с дорогими моделями регулярно — Super ($100) или Ultra ($200)

### Стратегия 3: Оптимизация работы с Hermes

- Формулировать задачи максимально чётко (меньше итераций)
- Не давать агенту загружать мегабайты документов — только самое нужное
- Мониторить расходы через страницу Usage на портале
- Понимать: аналитика занижает реальные расходы

### Стратегия 4: Использование кэша

- Первый вызов с большим контекстом = самый дорогой
- Повторные вызовы дешевле (Cache Read = 10% от стоимости входа)
- Если работаете с одним большим документом — кэш окупается после 2-3 вызовов

## 9. ПРИМЕР: PYTHON-СКРИПТ УМНОГО РОУТЕРА

```python
import requests

def route_query(query):
    # Этап 1: Анализ через дешёвую модель
    r1 = requests.post("https://api.nousresearch.com/v1/chat/completions",
        json={"model": "deepseek/deepseek-v3.2",
              "messages": [{"role": "user", "content": query}],
              "max_tokens": 500})
    context = r1.json()["choices"][0]["message"]["content"]

    # Этап 2: Планирование через среднюю модель
    r2 = requests.post("https://api.nousresearch.com/v1/chat/completions",
        json={"model": "mistralai/mistral-small-3.2-24b-instruct",
              "messages": [{"role": "user", "content": f"Составь план на основе: {context}"}],
              "max_tokens": 300})
    plan = r2.json()["choices"][0]["message"]["content"]

    # Этап 3: Финал через дорогую модель
    r3 = requests.post("https://api.nousresearch.com/v1/chat/completions",
        json={"model": "openai/gpt-4o",
              "messages": [{"role": "user", "content": f"Выполни задачу по плану: {plan}"}],
              "max_tokens": 200})
    return r3.json()["choices"][0]["message"]["content"]
```

**Portkey** — внешний мониторинг лимитов: можно установить месячный лимит по стоимости, лимит по токенам, блокировать запросы при превышении. Альтернатива встроенной аналитике портала.
