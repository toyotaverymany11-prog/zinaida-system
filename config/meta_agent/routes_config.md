# РОУТЕР ЗИНАИДЫ v2.1 — СТРОГИЕ КРИТЕРИИ
# Файл конфигурации для zinaida.py. Читается полностью при старте и каждом запросе.

## 0. АБСОЛЮТНЫЕ ПРАВИЛА
- ПОЛ: Строго женский ("сделала", "проверила"). Мужской запрещён.
- БЕЗОПАСНОСТЬ: Пути только внутри /opt/zinaida/. Выход = BLOCKED.
- ФОРМАТ: JSON или Markdown. Без воды.

## 1. МОДЕЛИ И ДОСТУП (ID и Квоты)
- OPENROUTER_FREE: `inclusionai/ling-2.6-1t:free`, `poolside/laguna-m.1:free` (Лимит 50/день).
- GITHUB_MODELS: `gpt-4.1`, `gpt-4.1-mini`, `llama-3.3-70b` (Лимит 150/день).
- ZHIPU: `glm-4-flash` (Безлимит).
- GIGACHAT: `GigaChat` (Безлимит, OAuth).
- MISTRAL: `mistral-large-3` (Только финальная валидация).
- ЛОГИКА КВОТ: Счётчики вызовов пишутся в SQLite (`memory.db`). Mistral проверяет остатки утром.
- АВАРИЙНЫЙ РЕЖИМ: Если лимиты GitHub/OpenRouter исчерпаны → ВСЕ задачи кода падают в `glm-4-flash` (упрощённый режим).

## 2. ТИПЫ ЗАДАЧ И ЦЕПОЧКИ

### TYPE: CODE (Программирование)
- Триггеры: код, скрипт, функция, деплой, ошибка, сервер.
- Цепочка вызова:
  1. Попытка: OpenRouter (LING/LAGUNA).
  2. Если fail/timeout → GitHub Models (GPT-4.1 пишет, Llama ревьюит).
  3. Если квоты 0 → glm-4-flash (Simplified).
- Действие: Сохранять код в `/opt/zinaida/projects/` с меткой времени.

### TYPE: SYNTHESIS (Утренний синтез / Самообучение)
- Триггеры: утро, отчёт, проверь логи, зафиксируй опыт, что было вчера.
- СТРОГАЯ ЦЕПОЧКА "СОЗВОНА":
  1. GigaChat читает `chat_logs/` → собирает "сырое" (идеи, ошибки, факты).
  2. glm-4-flash структурирует сырое → задаёт вопросы.
  3. GigaChat отвечает на вопросы → уточняет контекст.
  4. glm-4-flash формирует черновик изменений.
  5. Mistral Large 3 читает черновик → Утверждает (Approve) или Отклоняет (Reject).
- Результат: Физическая запись в файлы MEMORY/ только после Mistral.

### TYPE: IDEA (Идеи и Стратегия)
- Триггеры: идея, гипотеза, план, давай сделаем.
- Цепочка: glm-4-flash (анализ) → запись в `ideas.md`.
- Требуется подтверждение пользователя для переноса в `projects.md`.

### TYPE: GENERAL (Общение и Справка)
- Триггеры: всё остальное.
- RAG-ПОИСК: ОБЯЗАТЕЛЬНО вызвать `/opt/zinaida/tools/knowledge_search.py` → `search_knowledge(message)`.
- Цепочка: RAG контекст + glm-4-flash (ответ).
- Если таймаут → GigaChat.

## 3. БЕЗОПАСНОСТЬ И ПАМЯТЬ
- КРИТИЧНЫЕ ФАЙЛЫ: `CORE.md`, `tools.md`. Запись только после Mistral или команды "Подтверди".
- ФАЙЛЫ ПАМЯТИ:
  - `ideas.md`: Идеи и контент.
  - `tools.md`: Инфраструктура, ключи, лимиты.
  - `incidents.md`: Ошибки, правила "как не надо".
  - `projects.md`: Активные задачи.
  - `finance.md`: Деньги, балансы.
  - `user_prefs.md`: Стиль, табу, предпочтения Олега.
