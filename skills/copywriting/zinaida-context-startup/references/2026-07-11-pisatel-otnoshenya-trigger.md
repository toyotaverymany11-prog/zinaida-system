# Context Trigger: «писатель отношения» (добавлен 11.07.2026)

## Сигнал
Когда Олег пишет в новый чат: «писатель отношения», «писатель отношение2», «писатель отношения новая сессия».

## Не путать (ОБНОВЛЕНО 12.07.2026)
- Единоличное слово-роль **«писатель»** или **«писатель N»** = КОМАНДА загрузить конвейер, сказать «Поняла. Какая тема?» и ждать тему. НЕ спрашивать «какая задача/что делаем».
- **«писатель отношения»** в новом часе = команда загрузить полный контекст проекта + конвейер, показать сводку, спросить тему.
- **Важно:** «Писатель» НЕ команда сразу писать пост. Это переезд контекста из чата в чат + команда ждать тему. Пост пишется ТОЛЬКО после того как тема названа.
- Уже в чате, Олег говорит «напиши пост» = запустить конвейер генерации (Исследование №1 → 11 шагов → Исследование №2 → Контроль качества → показать)

## Протокол загрузки (6 шагов)
1. Читать HANDOVER (`/opt/zinaida/HANDOVER_TO_NEW_CHAT.md` если есть)
2. Читать живой слепок (`/opt/zinaida/memory/SYSTEM_SNAPSHOT.md`)
3. Проверить роутеры (curl 8002/8003/8005 → 200)
4. Прочитать pisatel-файлы (`/opt/zinaida/projects/Otnoshenya/pisatel/`)
5. SQL-запросы во все 4 БД (phases.db, smm_rag.db, content_rotation.db, analytics.db)
6. Запустить архитектурные скрипты: compile_writer_context.py + post_architect.py
7. Показать сводку со статусами ✅
8. Только после сводки спросить «пишем пост?»
