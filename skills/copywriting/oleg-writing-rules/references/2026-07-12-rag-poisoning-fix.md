# RAG-отравление: как старые черновики портили генерацию постов

## Проблема (обнаружена 12.07.2026)

При написании постов DeepSeek через 8005 роутер получал в контекст старые черновики из `/opt/zinaida/memory/` (Server RAG). Черновики содержали:
- Перечисление действий читательницы («она берёт телефон», «она пьёт кофе»)
- Паттерн «это не X, это Y» 
- Поп-психологию («тело чувствует», «холод в груди»)
- Устаревшие структуры из старых версий 01_style_scheme.md

DeepSeek воспринимал эти файлы как релевантный контекст и воспроизводил их стиль в ответе.

## Файлы-отравители

Перемещены в `/opt/zinaida/memory/archive/`:
- `post_vk_izmena_draft.txt` — черновик поста про измену (содержал перечисления её действий)
- `post_architecture_latest.txt` — старый шаблон структуры
- `writer_context_latest.txt` — старая версия контекста писателя
- `server_research_latest.txt` — старый результат исследования
- `MEMORY.md` — старый системный файл
- `BASELINE.txt` — старый baseline

## Системные фиксы

### 1. Glob-исключения в rg (router_8005_v2.py, строка 252)
Добавлены:
```python
"--glob", "!archive/*",
"--glob", "!*draft*",
"--glob", "!*.bak",
```

### 2. Приоритетная папка writer_rag
Создана `/opt/zinaida/writer_rag/` с проверенными файлами:
- `14_WRITER_FORMATION.md` — мастер-документ (единственный источник правил)
- `02_blacklist.md` — чёрный список
- `oleg_writing_rules_master.md` — все 16 комментариев Олега
- `approved_phrases.md` — одобренные фразы-эталоны

Добавлена первой в search_dirs роутера (приоритетный поиск).

### 3. Дополнительная чистка
- В `rg` команду добавлены исключения `!archive/*`, `!*draft*`, `!*.bak`
- В `find` команду (поиск по имени) также исключены `*.bak`, `*backup*`, `*cache*`

## Проверка

```bash
# RAG больше не видит черновики
rg -i -l "измен\|любовниц" /opt/zinaida/memory/archive/ 2>/dev/null  # → пусто

# RAG видит правильные файлы
rg -i -l "правило\|запрет" /opt/zinaida/writer_rag/ 2>/dev/null
# → 14_WRITER_FORMATION.md, oleg_writing_rules_master.md, 02_blacklist.md
```
