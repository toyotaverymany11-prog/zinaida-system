# Project Folder Approach — проектная файловая память

## Проблема

MCP-память (agentmemory, Zinaida Memory) ненадёжна между сессиями Hermes:
- **agentmemory MCP** — при каждом старте новой сессии создаёт пустую in-memory БД. Данные на диске есть (`standalone.json`), но MCP не подхватывает их.
- **Zinaida Memory** — persistent через SQLite, но поиск по `query` может не найти записи если формулировка не совпадает, а поиск по `tags` требует точного совпадения.
- **session_search** — работает стабильно, но ограничен текстом сессий (не структурирован).

## Решение: папка проекта на файловой системе

Вместо MCP-памяти — **читаемая агентом папка с markdown-файлами.**

### Концепция

```
/opt/zinaida/projects/<ProjectName>/<topic>/
├── 00_README.md          ← выжимка (самое важное на 1 страницу)
├── 01_registry.md        ← полная история тестов/изменений
├── 02_detail.md          ← технические детали
├── 03_feedback.md        ← оценки, критика, пожелания
└── 04_approved/          ← утверждённые файлы (с README со ссылками)
    └── README.md
```

### Почему это работает

1. **Надёжно:** файлы на диске не сбрасываются между сессиями
2. **Предсказуемо:** агент читает все файлы → получает полный контекст
3. **Прозрачно:** Олег может открыть папку и увидеть что есть
4. **Расширяемо:** можно добавить новый файл в папку — агент прочитает
5. **Без MCP:** не зависит от `npx -y @agentmemory/mcp`, env переменных, persistence настроек

### Структура для проекта «Отношения» / дизайн

```
/opt/zinaida/projects/Otnoshenya/design/
├── 00_README.md         — 88 строк: правила Олега, референс, 3 направления, цвета, шрифты, пути
├── 01_registry.md       — 42 строки: история тестов 7 моделей, утверждённые файлы
├── 02_models_tested.md  — 68 строк: каждая модель детально, три направления
├── 03_feedback.md       — 38 строк: 7 записей оценок Олега по датам
└── 04_approved/
    └── README.md        — 35 строк: таблица утверждённых файлов с путями
```

### Как это интегрировано

1. **AGENTS.md** — протокол загрузки начинается с ШАГ 0: прочитать всю папку проекта
2. **built-in memory** — содержит краткий протокол (подгружается в новый чат автоматом)
3. **zinaida-operations skill** — обновлён раздел старта с приоритетом файловой папки

### Как добавить новый проект

```bash
mkdir -p /opt/zinaida/projects/<ProjectName>/<topic>/04_approved
# Создать 00_README.md с:
# - железные правила
# - ключевые решения
# - ссылки на другие файлы в папке
# Скопировать структуру из Otnoshenya/design/ как шаблон
```

### Ограничения

- Агент должен явно читать файлы при старте (не автоматически, не как built-in memory)
- Если файлов много — чтение всех может занять несколько tool calls
- Не подходит для данных, которые должны обновляться автоматически (только ручные правки)
