# Сессия 2026-07-08 — фикс Telegram бота

## Проблема
Пользователь (Олег) сообщил: «в Telegram ты не читаешь, не отвечаешь, надо чтоб фото/видео/голосовые всё принимала»

## Диагностика

1. Сервис `zinaida-telegram-bot.service` — active (running), poll-ит getUpdates каждые 10с
2. Логи: `ERROR Connection error: Cannot connect to host 127.0.0.1:8644`
3. `curl http://127.0.0.1:8642/health` — `{"status": "ok"}` (Hermes жив на 8642)
4. `ss -tlnp | grep 864` — Hermes gateway на 8642, Web UI на 8648. 8644 НЕТ
5. Код bot.py: `HERMES_API_URL = "http://127.0.0.1:8644"` — неверный порт
6. Хендлер сообщений: только `filters.TEXT & ~filters.COMMAND` — медиа игнорируются

## Фикс

### 1. Исправлен порт Hermes API
```python
# Было:
HERMES_API_URL = "http://127.0.0.1:8644"
# Стало:
HERMES_API_URL = "http://127.0.0.1:8642"
```

### 2. Добавлена обработка медиа
Добавлены хендлеры для всех типов вложений:
```python
app.add_handler(MessageHandler(filters.PHOTO, handle_media))
app.add_handler(MessageHandler(filters.VIDEO, handle_media))
app.add_handler(MessageHandler(filters.VOICE, handle_media))
app.add_handler(MessageHandler(filters.Document.ALL, handle_media))  # НЕ filters.DOCUMENT!
app.add_handler(MessageHandler(filters.AUDIO, handle_media))
```

Написана функция `handle_media()`:
- Определяет тип (photo/video/voice/document/audio)
- Скачивает через `file_obj.download_as_bytearray()`
- Сохраняет в `/opt/zinaida/telegram_bot/media/` с UUID-именем
- Передаёт Зинаиде описание + пути к файлам + подпись пользователя

### 3. Добавлен `handle_media()` с загрузкой файлов
```python
async def download_media(file_obj, file_name):
    file_path = os.path.join(MEDIA_DIR, file_name)
    file_bytes = await file_obj.download_as_bytearray()
    async with aiofiles.open(file_path, "wb") as f:
        await f.write(file_bytes)
    return file_path
```

### 4. Сообщение о медиа строится так:
```
[Медиа: /opt/zinaida/telegram_bot/media/photo_abc12345.jpg] (photo)
📸 Фото (1920×1080)
Подпись: смотри что прислали
```

## Затык при деплое

При первом перезапуске сервис упал:
```
AttributeError: module 'telegram.ext.filters' has no attribute 'DOCUMENT'
```

В python-telegram-bot 22.x фильтр документов — `filters.Document.ALL`, а НЕ `filters.DOCUMENT`.
