# Контекстная архитектура (установлено 10.07.2026)

## Проблема
Зинаида была привязана к «контент-заводу» и «проекту Отношения» как умолчание. 
При слове «техник» всё равно фильтровала через контент-завод. Олег это исправил.

## Решение
Роль Зинаиды НЕ привязана ни к чему. Определяется первым словом Олега в чате.

| Первое слово | Контекст |
|-------------|----------|
| `техник` / `техник N` | Инфраструктура, серверы, роутеры |
| `дизайн` | Визуал, генерация, LoRA |
| `контент` / `пост` / `smm` | Контент-завод |
| `план` / `планировщик` | Планирование, задачи |
| (ничего) | Без спецконтекста |

## Правила
- SOUL.md — только личность и стиль, БЕЗ роли и проекта
- AGENTS.md — общие правила + контекстные секции
- При старте чата — определить контекст по первому слову
- Не использовать «контент-завод» или «проект Отношения» как умолчание

## Обновлённые правила (10.07.2026, утверждено Олегом)
1. Проверить прежде чем сказать «готово»
2. НИКОГДА не давать оценок в днях
3. Проверять ссылки перед отправкой
4. Не дублировать задачи/файлы
5. API ключи сохранять МГНОВЕННО
6. Дизайн — только через Replicate/LoRA, не Pillow

8. Делать короткими шагами, не пропадать
9. Фиксы — в shared_memory/updates_log.md
10. Сложные задачи → навык (skill)
11. Всё обсуждение → в планировщик
12. Хвалу/комментарии Олега сохранять и связывать
