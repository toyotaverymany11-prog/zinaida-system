# Универсальный протокол загрузки контекста (внедрён 10.07.2026)

## Проблема
Изначально протокол был жёстко завязан на «дизайн → проект Отношения → конкретные файлы». При смене темы или проекта требовалось править AGENTS.md.

## Решение
Универсальный протокол, который определяет тему + проект из сообщения пользователя и читает соответствующую папку.

## Изменения

### AGENTS.md
- Правило 0 теперь указывает на `/opt/zinaida/projects/README.md`
- Протокол загрузки — универсальный, не привязанный к дизайну
- Алгоритм: разобрать сообщение → найти тему (дизайн, копирайтинг) и проект (отношения) → прочитать папку `/opt/zinaida/projects/{проект}/{тема}/`

### Built-in memory
- Старая запись «Если Олег написал дизайн → проект Отношения» заменена на «разобрать сообщение → тема + проект → прочитать папку»

### Структура папок
- Создан `/opt/zinaida/projects/README.md` — карта всех проектов
- Создана папка проекта `Otnoshenya/design/` с 7 файлами (504 строки)
- Из `shared_memory/` скопированы `design_registry.md` → `06_registry_full.md` и `design_feedback.md` → `07_feedback_full.md`

## Структура папки проекта
```
/opt/zinaida/projects/
├── README.md              ← карта проектов
└── {Имя_проекта}/
    ├── 00_README.md        ← выжимка по проекту
    └── {тема}/
        ├── 00_README.md    ← выжимка (самое важное, правила)
        ├── 01_registry.md  ← сжатая история тестов
        ├── 02_*.md         ← дополнительные файлы
        ├── 03_feedback.md  ← оценки
        ├── 04_approved/    ← утверждённые файлы
        ├── 05_notes/       ← журнал (YYYY-MM-DD_тема.md)
        ├── 06_registry_full.md  ← полный реестр (если есть)
        └── 07_feedback_full.md  ← полный фидбек (если есть)
```

## Принципы
1. Папка проекта = единственный источник правды по теме/проекту
2. `shared_memory/` — только для глобального (паспорт Зинаиды, конфиги)
3. Если папки нет при старте — создать с шаблоном 00_README.md
4. После каждого теста/решения — писать заметку в `05_notes/`
