# Startup Protocol Overhaul (2026-07-10)

## Проблема: в новом чате протокол не выполнился

При старте нового чата с сообщением «дизайн отношения» агент:
- ❌ Не проставил ✅ галочки по источникам
- ❌ Не дал мнение/анализ
- ❌ Не отфильтровал по проекту
- ❌ agentmemory показал 0 записей (MCP стартует пустым)
- ❌ Zinaida Memory показал 0 записей (тег "проект:отношения" не был установлен)

## Причины

1. **Протокол был информацией, не командой.** В AGENTS.md и memory были инструкции, но они не форсировали выполнение.
2. **agentmemory MCP теряет данные между сессиями.** MCP сервер стартует с пустой in-memory БД, не подхватывая `/root/.agentmemory/standalone.json` (где лежат 5 записей).
3. **Zinaida Memory не имела тега "проект:отношения".** Поиск по `tags=["проект:отношения"]` давал 0, хотя записи были.

## Фиксы

### 1. AGENTS.md — полный пересмотр
- Добавлен раздел «ПРОТОКОЛ СТАРТА НОВОГО ЧАТА — ОБЯЗАТЕЛЕН К ИСПОЛНЕНИЮ» сразу после правила 0
- Каждый шаг — с ✅ форматом и конкретным инструментом
- Добавлен ШАГ 6 — МОЁ МНЕНИЕ (анализ + план)
- Добавлено определение проекта из сообщения
- Протокол теперь с формулировкой «ВЫПОЛНИТЬ», а не «загружаю контекст»

### 2. agentmemory — env переменные для persistence
В `/root/.hermes/config.yaml` добавлено:
```yaml
env:
  AGENTMEMORY_STORAGE_PATH: /root/.agentmemory/standalone.json
  MEMORY_FILE_PATH: /root/.agentmemory/standalone.json
```
Данные на диске есть (5 записей), но MCP может не подхватить — проверить в новом чате.

### 3. Zinaida Memory — тег "проект:отношения"
Сохранено 3 записи с тегом `проект:отношения`:
- ID 27: реестр дизайна, шрифты, цвета
- ID 28: тесты моделей, утверждённые файлы
- ID 29: три направления, правила Олега

### 4. built-in memory — лимит увеличен до 50000
Было 20000 (99% заполнено), стало 50000.

```

МОЁ МНЕНИЕ:
[Анализ опыта, что сработало/нет, рекомендация, план]
```
