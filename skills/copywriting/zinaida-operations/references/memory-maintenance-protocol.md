# Memory Maintenance Protocol (12.07.2026)

## Когда применять
Когда Hermes memory (быстрая память профиля, memory_char_limit в config.yaml) заполнена ~95%+.

## Правильный порядок
**Сначала почистить → потом расширить лимит.** Не наоборот — если сразу расширить, мусор останется.

## Шаги
1. Оценить записи: какие устарели (однократные задачи, детали уже в навыках)
2. Удалить через `memory(action='remove', old_text='уникальный фрагмент', target='memory')`
3. Цель — ~50-60% занятости
4. Расширить лимит: `hermes config set memory.memory_char_limit 10000`
5. Добавить новую запись

## Что удалять
- Детали выполненных одноразовых задач (если сохранены в навыках)
- Устаревшие инструкции, заменённые новыми правилами
- Технические детали разовых аудитов (они в навыках и references)

## Что НЕ удалять
- Правила обещаний (железное, критично)
- ДНК Зинаиды и стилевые принципы
- Текущие активные фиксы и конфигурации

## Пример из 12.07.2026
Было: 97% (4,878/5,000), 13 записей.
Удалены: "НЕ начинать работу по первому слову" (отменено триггером), "протокол аудита мёртвых systemd" (в навыке), "Техник 8 подробности" (в навыке), "Обновлены навыки после Техник 8" (избыточно).
Стало: 52%, 8 записей. Лимит расширен до 10,000.
