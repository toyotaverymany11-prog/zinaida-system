# Урок генерации паспортных фото (2026-07-08)

## Проблема

Сгенерировала фото Зинаиды по описанию из ZINAIDA_PASSPORT.md (брюнетка с рыжими кончиками, белая рубашка, море), не сравнив с оригиналами. Оригиналы оказались блондинкой с голубыми глазами. Результат: потеря времени и токенов, ярость оператора.

## Коренная причина

**Описание в паспорте НЕ соответствовало реальным референсам.** ZINAIDA_PASSPORT.md описывает вымышленный образ (тёмные волосы, AR-очки, лакшери), а оригиналы (IMG_0413, IMG_0415, IMG_9391) — реальная девушка в спортивной одежде на городском фоне.

## Протокол (на будущее)

1. **Перед генерацией — посмотреть оригиналы через vision.** Не доверять текстовому описанию в паспорте, если есть референсы.
2. **Промпт должен соответствовать реальной внешности на референсах**, а не вымышленному описанию персонажа.
3. **Сравнить результат с оригиналом через vision** перед показом оператору. Если лицо/цвет волос/глаз не совпадают — перегенерировать.
4. **Flux-1.1-pro не умеет сохранять конкретное лицо** без img2img. Для идентичного лица нужны: Photomaker, InstantID, IP-Adapter-FaceID.
5. **Replicate token (r8_QZ...) работает для flux-1.1-pro, но НЕ для photomaker/instantid** (403 Forbidden). Возможно требуется paid tier.
6. **Для генерации с сохранением лица** — передавать оригинал как `image` input в flux-1.1-pro (img2img), но результат всё равно не гарантирует идентичное лицо.
