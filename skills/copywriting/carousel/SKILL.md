```bash

echo 'export FAL_KEY=8e995491-ebb0-4650-8f66-dd0c2dee09ef:4fb8fc63694193ad8463f04ad9b25c1f' >> ~/.profile
```

**Теперь генерировать ЧЕРЕЗ `image_generate` tool, а не через curl.** Это штатный инструмент Hermes, быстрее и без ручной очереди.

```python
image_generate(aspect_ratio="portrait", prompt="...")
```

Только если `image_generate` снова сломается — fallback на curl (эндпоинт queue.fal.run).

**Шаг 1 — изменить модель в config.yaml через terminal (Python):**
```python
python3 -c "
import yaml
with open('/root/.hermes/config.yaml') as f:
    cfg = yaml.safe_load(f)
cfg['image_gen']['model'] = 'fal-ai/gpt-image-2'
with open('/root/.hermes/config.yaml', 'w') as f:
    yaml.dump(cfg, f)
"
```
(После этого нужно перезапустить Gateway: `npx hermes gateway start --daemon` в background)

**Шаг 2 — генерировать через curl:**
```bash
FAL_KEY="8e995491-ebb0-4650-8f66-dd0c2dee09ef:4fb8fc63694193ad8463f04ad9b25c1f"

# Отправить
curl -s -X POST https://queue.fal.run/fal-ai/gpt-image-2 \
  -H "Authorization: Key ${FAL_KEY}" \
  -H "Content-Type: application/json" \
  -d '{"prompt": "...", "image_size": {"width": 1080, "height": 1350}}'
# Ответ: {"status":"IN_QUEUE","request_id":"...",...}
```

## АРХИТЕКТУРА КАРУСЕЛИ (Instagram 5 Pillars)

### Pillar 1: Хук (Слайд 1)
- **80% всей вовлечённости** — решает судьбу карусели
- Время на реакцию: 0.25 секунды
- **Open Loop (главный нарративный приём 2026):** вопрос/интрига на слайде 1 → ответ на слайде 5-6. Создаёт психологическое напряжение — пользователь не может не досмотреть
- **Визуальные подсказки (swipe cues):** на правом краю слайдов 1-9 размещать стрелку → или часть изображения, «выпадающую» за край. Повышает swipe-through rate на 15-22%
- **≤40 символов (8-10 слов)** — строго. Больше = не прочитают в ленте
- Формулы: цифра-удар / вопрос / ошибка / трансформация

### Pillar 2: История (Слайды 2-8)
- 2: Контекст — «почему это важно»
- 3-4: Цифры + метафоры
- 5-6: Механика — «что происходит на самом деле»
- 7: «Честный минус» — почему просто «поговорить» не работает
- 8: Разворот — «а теперь самое интересное»

### Pillar 3: Инструмент (Предпоследний слайд)
- Детектив (УЛИКИ) / Телепат (СЦЕНАРИЙ) / Тренажер (ТРЕНАЖЁР)
- Радар (КАРТА) / Коллекция (ПАТТЕРНЫ) / Разобраться (РАЗБОР)
- Один пост = одна боль = один инструмент

### Pillar 4: Интерактив (Предпоследний или последний)
- «Save this for later»
- «Tag someone who needs this»
- «Send to a friend who...»

### Pillar 5: CTA (Последний слайд)
- Явный CTA даёт +20-30% вовлечённости
- Один конкретный призыв, не размытый
- Open Loop на следующий пост (анонс)

## АЛГОРИТМ INSTAGRAM 2026 (сигналы по приоритету)

### Андромеда AI — новый движок Meta
- **56% успеха = качество креатива** (данные Meta). Таргетинг, бюджет, время — в сумме весят меньше

- **Карусели = «горячий формат»** для Андромеды (много вариаций внутри одного поста)
- Распознаёт вариативность: 5 изображений с разным текстом на одинаковом фоне = одно сообщение. Штраф за монотонность (CPM растёт)

### Instagram = поисковая система (2026)
- Caption + on-screen текст сканируются алгоритмом для классификации контента
- Публичные посты индексируются Google
- **Хэштеги: макс 5, потеряли силу** (было 30). Общие теги не дают преимуществ
- Писать caption как мини-блог: раскрыть тему, встроить семантические ядра (поисковые запросы ЦА)
- **Тестируется**: индивидуальные заголовки к каждому слайду — позволит SEO-оптимизировать слайды отдельно

### Per-surface ranking: 4 алгоритма в одном
Instagram разделился на 4 независимых AI-driven ranking systems:
| Поверхность | Ключевые сигналы |
|-------------|------------------|
| **Feed** | Dwell time, saves, shares, completion rate |
| **Reels** | Продолжительность просмотра, % завершения, повторы, DM-шары |
| **Stories** | Близость к автору (DM, реакции), регулярность |
| **Explore** | Быстрый рост вовлечённости, популярность, история |

### Оригинальность (Originality Score)
- **10+ репостов чужого контента за 30 дней = полное исключение из рекомендаций** (Explore, Reels, Feed)
- Авторы оригинала видят +40-60% охвата
- Стратегия: только свой контент, уникальные инсайты

### Сигналы по приоритету
1. **Saves** — ×3 вес. Контент-«справочник» (чеклисты, списки, инструкции)
2. **Shares / DM Sends** — самый мощный сигнал. Одна пересылабельная фраза на пост
3. **Dwell Time** — секунды на посте, скорость свайпов
4. **Completion Rate** — >60% = Explore
5. **Swipe-Through Velocity** — нет dead stops
6. **Comments длинные (4+ слов)** — реальный диалог. Короткие «спасибо» почти не учитываются
7. **Likes** — почти не влияют
