# Zinaida → Лера: Workflow для генерации обложек

## Источник задачи
Задачи приходят из файлов вида:
```
/opt/zinaida/shared_memory/lera_research/TASK_*.md
```

Они содержат:
- Формат (напр. 1590×400 px)
- Модель (обычно `ideogram-ai/ideogram-v3-turbo` — единственная с правильной кириллицей)
- Цветовую гамму (тёплая, бургунди/золото — не чёрный, не плоский)
- Варианты обложек (1/2/3) с готовыми промптами
- Путь сохранения

## Ключевые правила

### 1. ОДНА КАРТИНКА ЗА РАЗ (не батч)
⚠️ Задание всегда явно указывает: «СДЕЛАЙ ПО ОДНОЙ КАРТИНКЕ НА ВАРИАНТ. НЕ БАТЧ.»
Генерируй по одному запросу, жди результат, потом следующий.

### 2. Модель — Ideogram V3 Turbo
- `ideogram-ai/ideogram-v3-turbo` — $0.03/img
- Указывай `model: "V_3_TURBO"` в параметрах
- Aspect ratio: только из пресетов. Для 1590×400 используй `"3:1"` (дает 1536×512), потом обрезай центром через Pillow.
- Кириллица: всегда добавляй в промпт усиление:
  ```
  Every letter perfectly legible: М-Е-Г-А-М-О-З-Г. Correct Russian Cyrillic spelling, all letters clearly readable.
  ```
- FileOutput: `output.url` для получения URL, `requests.get()` для скачивания

### 3. Пост-обработка
После генерации:
```python
img = Image.open(intermediate_path)
w, h = img.size
target = (1590, 400)
left = (w - target[0]) // 2
top = (h - target[1]) // 2
cropped = img.crop((left, top, left + target[0], top + target[1]))
# Сохранить как JPG quality=95
```

### 4. Пути сохранения
- Готовые изображения: `/opt/zinaida/design/generated/lera_test/test_<variant>.jpg`
- Временные файлы удалять после обработки

### 5. DONE.md
После генерации обязательно записать отчёт в `/opt/zinaida/design/generated/lera_test/DONE.md`:
- Дата
- Модель
- Использованный промпт
- Aspect ratio / crop
- Результат (путь, размер, вес)
- Результаты проверки (кириллица, цветовая гамма, элементы композиции)

### 6. Проверка качества
После сохранения — проверить результат через vision_analyze:
- Корректность кириллицы
- Наличие всех элементов из промпта
- Соответствие цветовой гамме
- legibility текста
