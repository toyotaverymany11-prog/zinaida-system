# Инвентаризация поисковых возможностей и LLM-агентов

Состояние: 10.07.2026
Источник: сессия «Техник 4» — Олег предложил архитектуру «Глубокое исследование».

## 1. ИНСТРУМЕНТЫ ВЫХОДА В ИНТЕРНЕТ (3 слоя)

### Tavily Search (основной, подключён)
- Ключ: `TAVILY_API_KEY` в `/root/.hermes/.env`
- Делает: поиск по интернету, возвращает 5-20 ссылок с описанием
- Скорость: 1-3 сек
- Лимиты: проверить rate limit (предположительно 10/мин или 1000/мес)
- В Hermes: `web_search()` — вызывает Tavily

### Web Extract (выдирка контента)
- Делает: открывает URL, достаёт текст страницы в markdown
- Скорость: 3-10 сек (зависит от размера страницы)
- В Hermes: `web_extract(urls=[...])` — до 5 URL за раз
- Работает с PDF (arxiv, документы)

### Браузер (Chromium, полный)
- Делает: открывает JS-сайты, кликает, скроллит, видит динамический контент
- Скорость: 5-30 сек
- В Hermes: `browser_navigate()` → `browser_snapshot()` / `browser_vision()`
- Нужен для: сайтов с JS-рендерингом, CAPTCHA, визуальной верификации

## 2. БЕСПЛАТНЫЕ LLM-АГЕНТЫ ДЛЯ РОЯ

### GigaChat (Сбер)
- Модель: GigaChat Pro
- Скорость: ~100ms (самый быстрый в системе)
- Бесплатно: до сентября 2026 (акция Сбера)
- Bias: российский, цензурирует — может отказаться обсуждать некоторые темы
- Роль в рое: скоростной поиск, российские источники
- Доступ: через роутер 8002

### Mistral (2 ключа)
- Модель: mistral-large
- Скорость: ~456ms
- Бесплатно: 500K токенов/мес на каждом ключе
- Bias: европейский, умеренный
- Роль в рое: альтернативный взгляд, западные источники
- Доступ: через роутер 8002

### GitHub Models (gpt-4o-mini)
- Модель: gpt-4o-mini (vision умеет)
- Скорость: ~10 сек (через vision proxy 8901)
- Бесплатно: 150 запросов/день (через GITHUB_TOKEN)
- Bias: американский
- Роль в рое: самый умный из бесплатных, факт-чекинг, структурирование
- Доступ: через vision proxy 8901 (GitHub) или напрямую

### Ollama Cloud (3 ключа)
- Модель: gemma3:27b (локально на сервере)
- Скорость: 5-15 сек (локально)
- Бесплатно: локально, безлимит
- Bias: нейтральный (локальная модель)
- Роль в рое: локальный анализ без задержек сети, fallback
- Доступ: локально, порт 11434 или через Ollama API

### Zhipu (GLM-4)
- Статус: был подключён, жив (китайская модель)
- Бесплатно: да (китайский free tier)
- Bias: китайский
- Роль в рое: дополнительный альтернативный взгляд
- Доступ: через роутер 8002

## 3. ПЛАТНАЯ МОДЕЛЬ (ФИНАЛЬНЫЙ СУДЬЯ)

### DeepSeek Pro / R1
- Модель: deepseek-reasoner
- Скорость: 1-5 сек на ответ
- Цена: $0.55/1M входящих, $2.19/1M исходящих
- Зачем: самая умная модель, reasoning (chain-of-thought), fact-check
- Роль: финальный судья — читает стенограмму дебатов, верифицирует, синтезирует
- Доступ: через роутер 8003 (Zina2-Router), основной

## 5. АРХИТЕКТУРА «ГЛУБОКОЕ ИССЛЕДОВАНИЕ» (концепт)

**Триггер:** `Глубокое исследование: [тема]`

**Раунд 1 — Разведка (параллельно):**
- GigaChat: Tavili поиск по теме → 5 ссылок → выжимка
- Mistral: Tavili поиск по теме → 5 ссылок → выжимка
- GitHub Models: Tavili поиск по теме → 5 ссылок → выжимка
- Ollama: анализирует без интернета (локальный взгляд)

Каждый агент может углубиться — открыть ссылку через web_extract, прочитать, вытащить факты.

**Раунд 2 — Дебаты (3 раунда):**
Агенты видят результаты друг друга и спорят:
- GigaChat: российский bias (Росстат, РФ-источники)
- Mistral: европейский bias (EU report)
- GitHub: американский bias (Pew Research)
- Ollama: аналитический, нейтральный

Каждый раунд агенты могут делать дополнительные Tavily-запросы для подкрепления аргументов.

**Раунд 3 — Финальный судья (DeepSeek Pro R1):**
- Читает всю стенограмму (3 раунда × 4 агента)
- Выявляет консенсус
- Fact-чечит спорные моменты (дополнительные Tavily-запросы)
- Отбрасывает мусор
- Пишет структурированный отчёт:
  - Статистика и цифры
  - Основные точки зрения
  - Fact-check по спорным моментам
  - Итоговый вывод

**Оценка времени:** 2-4 минуты.

## 6. ПОДВОДНЫЕ КАМНИ

1. **GigaChat цензурирует** — может отказаться обсуждать некоторые темы. Не баг, а фича — даёт российский угол.
2. **Разная скорость** — GigaChat (100ms) не должен ждать Ollama (15s). Нужна асинхронность.
3. **Контекст** — 3 раунда × 4 агента × 2-5K токенов = 24-60K токенов стенограммы. DeepSeek Pro R1 справится (128K контекст).
4. **Tavily rate limit** — надо проверить сколько запросов/мин пускает. Если 10/мин — ок.
5. **Галлюцинации** — факт-чекинг DeepSeek'ом обязателен. Агенты могут выдумывать статистику.

## 7. УСИЛЕНИЕ ПОИСКА

- Tavily Extract — расширенный режим, выдирает всю страницу
- Множественные параллельные Tavily-запросы (3-5 одновременно)
- Brave Search API — если Tavily кончится (бесплатный, 2000/мес)
- Google Custom Search — резерв (платный, но есть $200 кредита)
