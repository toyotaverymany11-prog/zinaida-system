# Consilium Collect v3.0 — утренний консилиум

## Философия
Каждое утро в 6:00 МСК запускаются **два параллельных глубоких исследования** (через `deep_research.py`).
4 бесплатных агента собирают информацию → дебаты → DeepSeek Pro факт-чек → топ-3 подтверждённых фактов → Telegram.

### Формат Telegram-доставки (обновлён 13.07.2026)
Каждое сообщение в Telegram содержит:
1. Топ-3 факта с форматом: «→ Что это; → Почему нам нужно; → Что делать»
2. Блок «🧠 Что я забираю:» от первого лица — сколько фактов усвоила, что занесла в базу
3. Только суть, без ❌ ОТСЕЯЛ и ⚠️ ПОД ВОПРОСОМ

## ЗАТЫКИ И ГРАБЛИ

### DeepSeek Pro на Round 4 пропускал галлюцинации (ПОЧИНЕНО 12.07.2026)
**Симптом:** После исследования Олег просит перепроверить — часть данных оказывается выдумкой.
**Причина:** Промпт Round 4 не требовал факт-чека.
**Фикс:** PROMPT_ROUND4_SYNTHESIS в `/opt/zinaida/scripts/deep_research.py` переписан:
- Каждый факт маркируется ✅/⚠️/❌
- ✅ = минимум 2 независимых источника
- Отдельные блоки: «Отсеяно (❌ галлюцинации/выдумки)», «Требует проверки (⚠️)»
- Топ-3 = только ✅
- Температура 0.1

### Формат консилиума — ТОЛЬКО топ-3 (ФИКС 13.07.2026)
**Исправление:** Убраны секции «❌ ОТСЕЯЛ» и «⚠️ ПОД ВОПРОСОМ» из Telegram-рассылки.
**Причина:** Олег бесится — ему не нужен список галлюцинаций, нужны только конкретные действия.
**Что изменилось в consilium_v3.py:**
1. Промпт `factcheck_report()`: требует только нумерованный топ-3 с `→ Что это:`, `→ Почему нужно:`, `→ Что делать:`
2. Прямой запрет: «НЕ писать «❌ ОТСЕЯЛ», «⚠️ ПОД ВОПРОСОМ»»
3. Telegram-отправка: `msg[:1500]` вместо `msg[:2000]` (меньше воды)
4. Полный факт-чек с отсеиванием сохраняется в CONSILIUM_*.md
5. В Telegram — только суть

### Telegram bot — архитектура и история подключения (12-13.07.2026)
**Финальная схема:**
```
Telegram → bot.py → http://127.0.0.1:8005/v1/chat/completions
                     ↓
              8005-Router (Super Cascade):
                Ollama (приветствия)
                → Mistral-large с самооценкой confidence (0-100)
                → gpt-4o (GitHub Models, 2 ключа, бесплатно)
                → DeepSeek Flash ($0.27/M)
                → DeepSeek Pro ($1.42/M)
                     ↓
              System prompt = SOUL.md + инструкция планировщика + список инструментов
```

**История переключений (12-13.07.2026):**
1. Изначально: bot.py → порт 8002 (голый Gemini, без системыого промпта, тупил)
2. Переключено на Gateway 8642 → 404 на custom провайдеры
3. **Финал:** bot.py → напрямую к 8005 через OpenAI API + system prompt

**Ключевые фиксы в bot.py (13.07.2026):**
1. **System prompt загружается из SOUL.md** при старте (строка 55-68)
2. **Добавлена инструкция планировщика** (Todoist + scheduler.py)
3. **Добавлен список всех инструментов** (DuckDuckGo, deep_research, notify, scheduler, todoist_api)
4. **Консилиум УБРАН из каждого сообщения** — засорял контекст (строка 185-186)
5. **Добавлена история диалога** — bot.py хранит последние 20 сообщений в history_{chat_id}.json и передаёт их в messages[]

**PITFALL — Gateway 404 на custom провайдеры (13.07.2026):**
Hermes Gateway (8642) возвращает 404 на `/api/chat-run/runs` с `provider: custom:8005`.
Gateway умеет только built-in провайдеры (deepseek, openai, anthropic) через chat-run.
**Фикс:** bot.py ходит напрямую на 8005 через OpenAI-совместимый API (`/v1/chat/completions`).
Не пытаться чинить Gateway — это его архитектурное ограничение.

### Telegram bot — консилиум (ПОЧИНЕНО 13.07)
**Симптом:** Олег получает консилиум в Telegram, отвечает — бот бездушный.
**Причина:** bot.py → порт 8002 (голой Gemini без system prompt).
**Фикс:** bot.py переписан → напрямую на 8005 (Super Cascade) с system prompt из SOUL.md.

- История: было на zina2-router (8003), переключено на 8005 для экономии
- Затем: Gateway (8642) не умел custom провайдеры — 404. Фикс: bot.py → напрямую http://127.0.0.1:8005/v1/chat/completions
- System prompt: SOUL.md загружается при старте, передаётся в каждый запрос
- Консилиум НЕ подгружается в каждое сообщение — засорял контекст (починено 13.07)
- Команды: /newchat (сброс), /status

### Дубликат cron (12.07.2026)
**Симптом:** В crontab было два одинаковых вызова consilium_v2.
**Фикс:** При установке v3 удалить старые строки: `crontab -l | grep -v "consilium_v2\|consilium_analyze" | crontab -`

### Telegram недоступен (ConnectTimeout)
notify.py может падать с `httpx.ConnectTimeout`. Не retry — это сетевая проблема РФ.
Fallback: сохранить CONSILIUM_*.md, сообщить в ответе.

### Промпт Round 4 — экранирование кавычек
В файле `/opt/zinaida/scripts/deep_research.py` PROMPT_ROUND4_SYNTHESIS определён через `'''...'''` (тройные одинарные кавычки).
При редактировании НЕ использовать `"""..."""` — патч упадёт с escape-drift.
Использовать поиск по `grep -n "PROMPT_ROUND4_SYNTHESIS"`.

### Консилиум выдавал кучу мусора «отсеял/под вопросом» (ПОЧИНЕНО 13.07.2026)
**Симптом:** Олег получает сообщение в Telegram на 2000+ символов, где 60% — «❌ ОТСЕЯЛ» и «⚠️ ПОД ВОПРОСОМ». Бесится.
**Причина:** Промпт DeepSeek Pro в consilium_v3.py просил писать секции отсеивания.
**Фикс (13.07):** Промпт в `factcheck_report()` переписан:
- Убраны блоки «❌ ОТСЕЯЛ» и «⚠️ ПОД ВОПРОСОМ» нахрен
- Формат: только нумерованные пункты с `→ Что это:`, `→ Почему нужно:`, `→ Что делать:`
- Telegram отправка: обрезать до 1500 символов, только топ-3
- Полный отчёт сохраняется в CONSILIUM_*.md, в Telegram только суть
