# Многоагентное глубокое исследование

## Концепция

При фразе «глубокое исследование: [тема]» — запускается рой из 4 параллельных LLM-агентов. Каждый использует свой bias (разные модели) и свой поисковый канал. После сбора информации агенты формулируют осмысленные вопросы, затем делают целевую добивку, и финальный синтез делает сильная модель.

**Killer feature:** агенты НЕ просто собирают и дебатят — они формулируют вопросы по существу (с конкретными ссылками на факты). Это отсеивает мусор на раннем этапе.

## Архитектура (4 раунда)

```
Round 1 — СБОР         4 агента параллельно → Tavili → LLM-конспекты
Round 2 — ВОПРОСЫ      Каждый формулирует 3 вопроса → DeepSeek фильтрует
Round 3 — ДОБИВКА      Целевой поиск под годные вопросы
Round 4 — СИНТЕЗ       DeepSeek Pro → отчёт с ✅/⚠️/❌
```

## Выбор агентов (проверено 10.07.2026)

**Живые провайдеры для прямых API-вызовов (не через роутер):**

| Агент | Модель | Когда использовать |
|-------|--------|-------------------|
| **Mistral** (3 ключа) | mistral-small-latest | Основной воркер. Быстрый, бесплатный. |
| **GitHub Models** | gpt-4o-mini | Аналитик. Vision умеет, бесплатно. |
| **DeepSeek V3** | deepseek-chat | ТОЛЬКО Раунд 4 (синтез) + Раунд 2 (оценка). Платный, умный. НЕ Flash. НЕ Раунд 3. |
| **Mistral (второй ключ)** | mistral-small-latest | Дополнительный bias для разнообразия мнений. |

**МЁРТВЫЕ (не использовать):**
- ❌ OpenRouter — 403 policy на все free-модели
- ❌ GigaChat — 400 на OAuth (ключ сломан)
- ❌ Zhipu — статус неизвестен

## Поисковые каналы

| Канал | Статус | Лимит |
|-------|--------|-------|
| **Tavily** (tavily_search) | ✅ ЖИВ | 1000 запросов/мес бесплатно, $8/1k затем |
| **Brave Search** | ❌ НЕТ КЛЮЧА | 2000 запросов/мес бесплатно — ждёт ключа |
| **Web Extract** (прямой HTTP) | ✅ ЖИВ | Безлимит, встроен |

## Прототип

**Файл:** `/opt/zinaida/scripts/deep_research.py`
- Читает тему из stdin или аргументов командной строки
- Запускает 4 агента параллельно (ThreadPoolExecutor, 30s timeout)
- Сохраняет результаты в `/opt/zinaida/sandbox/deep_research/{timestamp}_{topic}/`
- Финальный отчёт: `final_report.md`

**Запуск:**
```bash
python3 /opt/zinaida/scripts/deep_research.py "тема исследования"
echo "тема" | python3 /opt/zinaida/scripts/deep_research.py
```

## Раунд 1 — Сбор

Каждый search-агент:
1. Tavily search (5 результатов, advanced depth, include_raw_content)
2. Для каждой ссылки — Web Extract (или берёт raw_content из Tavily)
3. LLM-конспект: 5-10 bullet points с цитатами и URL

**De-dup:** URL-адреса дедуплицируются (первые 3 агента могут найти одни и те же).

## Раунд 2 — Вопросы

Каждый агент получает все 4 конспекта и формулирует 3 вопроса.

**Критерии годного вопроса:**
- Содержит конкретную ссылку на факт/цифру из конспекта
- Требует углублённого поиска (не «расскажи подробнее»)
- Сформулирован как поисковый запрос

**Фильтрация:** DeepSeek Pro оценивает все вопросы (12 штук). ✅ годные / ❌ мусор.

## Раунд 3 — Добивка (параллельно, ThreadPoolExecutor)

Под каждый годной вопрос — целевой Tavily поиск. Ответ даёт **Mistral** (бесплатно), не DeepSeek. Дедупликация вопросов. Параллельная обработка до 5 вопросов одновременно.

## Раунд 4 — Синтез

DeepSeek Pro читает: конспекты (Round 1) + вопросы и оценки (Round 2) + ответы (Round 3).

**Формат отчёта:**
```
# Глубокое исследование: [тема]
Дата: ...
Источники: [N]

## 1. Ключевые выводы
...

## 2. Факты и цифры
- [факт] — ✅ [ссылка]
- [факт] — ⚠️ [ссылка] (требует проверки)
- [спорный момент] — ❌ [почему]

## 3. Анализ противоречий
...

## 4. Итоговый вердикт
...
```

## Подводные камни и фиксы

| Проблема | Симптом | Фикс |
|----------|---------|------|
| Агент упал | dead agent в отчёте | Graceful degradation — null, DeepSeek пишет «агент N не ответил» |
| Таймаут | агент не уложился в 30s | deadline в ThreadPoolExecutor + as_completed timeout |
| Tavily rate limit | пустые результаты | Уменьшить max_results или переключить тему |
| DeepSeek контекст переполнен | обрезанный отчёт | Summaries вместо полных логов (bullet points, 5-10) |
| Дублирующиеся вопросы | 4 агента спросили одно и то же | Дедупликация в parse_approved_questions |
| OpenRouter 403 | пустой агент | Заменить на другой провайдер (Mistral/DeepSeek) |

## Когда НЕ использовать

- Простые фактологические запросы («когда родился Пушкин») — хватит одного Tavily
- Задачи с жёстким дедлайном (<1 минуты) — 4 раунда занимают 90-180 секунд
- Темы, где нет публичных источников — внутренние корпоративные данные
