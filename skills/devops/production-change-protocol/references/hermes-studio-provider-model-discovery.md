# Hermes Studio: Provider-Model Discovery

## Как Hermes Studio отображает провайдеров и моделей

### Источник данных
Hermes Studio читает провайдеров из config.yaml **выбранного профиля**. Не из корневого конфига (providers), а из profile-specific `custom_providers`.

```
/root/.hermes/
├── config.yaml                          # Корневой конфиг (для default профиля)
│   └── providers:                       # ← Используется ТОЛЬКО default
│       - zinaida-router: {api, api_key, default_model, name}
│
└── profiles/
    └── zinaida/
        └── config.yaml                  # ← Используется для профиля zinaida
            └── custom_providers:         # ← Профильные провайдеры
                - name: 8005
                  base_url: http://127.0.0.1:8005/v1
                  model: 8005-Enhanced
```

### Как происходит отображение

1. **Чтение custom_providers** — Hermes Studio читает секцию из config.yaml профиля
2. **Группировка по base_url** — одинаковые base_url группируются в один блок UI
3. **GET {base_url}/v1/models** — для каждого уникального base_url делается запрос
4. **Отображение** — имя провайдера (name) как заголовок секции, модели из API как список

### Что определяет какие модели видны

| Куда смотреть | Что даёт |
|---------------|----------|
| `custom_providers[].model` | Модель по умолчанию (выделена в UI) |
| `{base_url}/v1/models` | Все доступные модели в секции |
| Поле "Пользовательская" в UI | Модели, введённые вручную (добавляются через `/api/hermes/custom-model`) |

### Как контролировать список моделей

**Единственный способ:** изменить ответ роутера на GET /v1/models.

**НЕ работает:**
- Добавление N провайдеров на один base_url — создаёт N секций в UI, а не N моделей в одной секции
- Добавление `model` как списка в custom_providers — невалидный синтаксис

### Проверка

```bash
# Что увидит Hermes Studio?
curl -s http://127.0.0.1:8005/v1/models

# Вывод:
# Список моделей, которые покажутся в интерфейсе под провайдером 8005
```

## История ошибок

### 13.07.2026 — 4 провайдера на 8005 вместо 1

**Было:**
```yaml
custom_providers:
  - name: router-8005-flash      # ← ОДИН base_url = ЧЕТЫРЕ строки в UI
  - name: router-8005            #   Каждая с одной моделью
  - name: router-8005-pro        #   Пользователь видит кашу
  - name: router-8005-enhanced
```

**Симптом:** В интерфейсе Hermes Studio появились 4 отдельных провайдера вместо одного с 2 моделями. Пользователь: «четыре блядь каких-то вариантов, у меня только два!»

**Фикс:** Один провайдер на base_url. Модели берутся из API роутера.
