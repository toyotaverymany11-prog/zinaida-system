# Чек-лист добавления LLM-агента в deep_research.py

## Обязательно проверить перед интеграцией любого нового агента:

1. **Модель должна быть БЕСПЛАТНОЙ** — проверить на сайте провайдера
   - Ollama: gemma3:4b или ministral-3:3b (НЕ gemma3:27b — платная)
   - GitHub: gpt-4o-mini (бесплатно, через GITHUB_TOKEN)
   - Mistral: mistral-small-latest (бесплатный лимит)
2. **Ключи загружены в load_config()** — после добавления переменной в .env, проверить что она читается в load_config()
3. **Тест прямого API-вызова** — вызвать через python requests ДО интеграции в скрипт
4. **Тест в скрипте** — запустить deep_research.py с новой темой, проверить что агент ответил
5. **Обработка ошибок** — если агент упал, должен быть fallback на другой (Mistral/DeepSeek)

## Известные грабли (НЕ повторять):
- gemma3:27b — платная модель ($), все запросы падают. Использовать gemma3:4b (бесплатно, ~2с)
- Ollama API ключи (OLLAMA_API_KEY_1/2/3) забыты в load_config() — проверять загрузку
- OpenRouter — IP сервера заблокирован (403), не использовать
- GigaChat — OAuth 400, не использовать
- **Маскировка ключей в terminal()** — если написать `token = "ghp_hX...v9o1"` в коде, то исполнится `token = "***"` и придёт 401. Ключи читать ТОЛЬКО через `open().read()` из .env. Подробнее: навык `devops/provider-audit-reference` раздел «Главный урок сессии: маскировка ключей».
- **DeepSeek Pro Round 4 (синтез) пропускал галлюцинации — ПОЧИНЕНО 12.07.2026.**
  Старый промпт PROMPT_ROUND4_SYNTHESIS не требовал факт-чека — DeepSeek просто писал красивый отчёт, даже если данные противоречили друг другу.
  **Фикс:** Новый промпт с жёсткой классификацией: ✅ (2+ источника) / ⚠️ (1 источник) / ❌ (нет источника).
  Отдельные блоки «Отсеяно» и «Требует проверки». Температура 0.1.
  При изменении промпта — убедиться что факт-чек остаётся обязательным.
