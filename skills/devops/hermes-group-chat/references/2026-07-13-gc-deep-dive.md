# 2026-07-13 — Group Chat Deep Dive

## Финальная архитектура: v7 (gc_agent_v7.py)

**Путь:** `/opt/zinaida/scripts/gc_agent_v7.py`
**systemd:** `hermes-gc-v7.service`

### Принцип работы
1. Один Socket.IO клиент, userId: "1" (super_admin)
2. Слушает `/group-chat` namespace через `@sio.on("*")`
3. На `@Ольга` или `@Зинаида` → запрос к 8003 (DeepSeek Flash)
4. Ответ пишет напрямую в `gc_messages` (INSERT INTO) — это надёжно, видно после F5
5. Дополнительно эмитит через `emit("message")` — на случай если сервер ретранслирует

### Почему emit("message") не работает
**Главная причина:** сервер Hermes Web UI v0.6.28/v0.6.29 не ретранслирует `emit("message")` от клиента другим клиентам.

Цепочка:
1. Клиент шлёт `emit("message", {...})` на сервер
2. Сервер вызывает `handleMessage(l, ...)` где l = сокет
3. `handleMessage` проверяет `canSocketManageRoom(l, room)` — ТОЛЬКО super_admin
4. Если проверка прошла → `nsp.to(roomId).emit("message", W)` (ретрансляция)
5. Если НЕ прошла → сообщение сохраняется в БД, но ретрансляции нет

Почему v7 пишет в БД напрямую: **emit может упасть молча, а БД-запись всегда сработает.**

### Что пробовано (все варианты)
| Вариант | Результат |
|---------|-----------|
| `source: "agent"` + wX | authMiddleware пропускает, handleJoin падает |

### Ошибка restoreAgents()
```
ReferenceError: t is not defined
    at JX.restoreAgents (../server/index.js:1568:5857)
```
- В v0.6.29 **та же ошибка**
- `restoreWhenReady()` вызывается, `restoreAgents()` падает
- Агенты в БД (`gc_room_agents`) есть, AgentClient не создаются
- **Единственный workaround:** внешний Python клиент

### Реальная проблема: архитектурное ограничение
Hermes Web UI Group Chat — это **демо/экспериментальная фича**:
- Нет REST API для отправки сообщений
- AgentClient создаётся только через Web UI (не через API)
- `restoreAgents()` падает с ReferenceError
- emit("message") от клиента не ретранслируется
- senderName нельзя подменить

**Для production нужно:** либо патчить server/index.js (8MB минифицированный, рискованно), либо переходить на другую платформу.
