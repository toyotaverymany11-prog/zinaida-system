# Как НЕ надо настраивать провайдеры в профилях

## Контекст: 13.07.2026, профиль zinaida

### Ошибка
Создал 5 custom_providers на один порт 8005:
- router-8005-enhanced
- router-8005
- router-8005-flash
- router-8005-pro
- zinaida-router

В Hermes Studio это отобразилось как 5 отдельных провайдеров с непонятными названиями, вместо группировки по base_url.

### Симптомы
- Захламлённый интерфейс выбора провайдера
- Непонятно что за роутеры и зачем
- Олег: «хуйню ебаную наделала», «не соответствует как у меня отображается»

### Правильное поведение
Один провайдер на один уникальный base_url. Модели подтягиваются из API роутера (GET /v1/models).

### Ключевой урок
Hermes Studio группирует custom_providers по `base_url`. Все провайдеры с одинаковым base_url → ОДНА секция в UI, внутри которой модели из API. Не надо создавать N провайдеров для N моделей одного роутера.

### Пример
```yaml
# ❌ НЕПРАВИЛЬНО — 4 провайдера на 1 порт
custom_providers:
  - name: router-8005-flash
    base_url: http://127.0.0.1:8005/v1
    model: 8005-Flash
  - name: router-8005-pro
    base_url: http://127.0.0.1:8005/v1
    model: 8005-Pro

# ✅ ПРАВИЛЬНО — 1 провайдер на порт
custom_providers:
  - name: 8005
    base_url: http://127.0.0.1:8005/v1
    model: 8005-Enhanced
```
