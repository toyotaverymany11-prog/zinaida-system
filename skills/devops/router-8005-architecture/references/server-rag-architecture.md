# Server RAG — поиск по файлам сервера по теме чата

## Концепция

8005 ищет информацию НА СЕРВЕРЕ по ТЕМЕ ТЕКУЩЕГО ЧАТА, а не в заранее заготовленной базе знаний.
Тема определяется из первого запроса в чате (ключевые слова).

## Два уровня

### Уровень 1: RAG-сборщик (быстрый, бесплатно)
- Инструмент: `ripgrep (rg)` — встроен в Linux
- Ищет: `.md`, `.py`, `.yaml`, `.json`, `.txt`, `.env` файлы в `/opt/zinaida/`
- Не ищет: `backup`, `cache`, `sessions`, `__pycache__`, `.git`, `node_modules`
- Результат: 3 куска по 500 символов, наиболее релевантные
- Время: 0.05 сек
- Промпт: извлекает ключевые слова из запроса, ищет их

### Уровень 2: Mistral-анализатор (умный, бесплатно)
- Модель: Mistral Small (через ротацию 3 ключей)
- Что делает: смотрит найденные файлы, ищет CROSS-СВЯЗИ
- Пример: файл `router_8005_v2.py` использует `DEEPSEEK_KEY` из `.env` — анализатор видит связь
- Результат: текстовое описание связей ("этот файл ссылается на тот, та проблема решается здесь")
- Время: 1-2 сек
- Timeout: 3 сек, если не успел — идём без него

### Параллельная архитектура

```
Запрос → Классификация
    │
    ├── Ollama (приветствия) ──→ ответ за 0.5 сек
    │
    └── DeepSeek ──→ параллельно:
         │              ├── RAG-сборщик (rg, 0.05 сек)
         │              └── Mem0 (факты по теме)
         │
         ├── Mistral-анализатор (1-2 сек, timeout 3 сек)
         │   └── если упал/таймаут → идём без него
         │
         └── DeepSeek Flash/Pro → генерация с контекстом:
              ├── куски файлов (RAG)
              ├── cross-связи (Mistral)
              └── факты (Mem0)
```

## Что НЕ делает

- НЕ ищет в интернете (для этого нужен Tavily)
- НЕ привязан к smm_rag.db (только для контент-завода)
- НЕ дублирует Mem0 (Mem0 = факты, RAG = файлы, Анализатор = связи)
- НЕ требует API-ключей — локальный rg бесплатен

## Проблемы и решения

| Проблема | Решение |
|----------|---------|
| Mistral упадёт 429 | Timeout 3 сек, не ждём — идём без анализа |
| Mistral придумает связи | Температура 0.1, строгий промпт |
| RAG найдёт мусор из backup/ | Исключаем: backup, cache, sessions, node_modules |
| На «привет» анализатор не нужен | Ollama отвечает сам, минуя остальное |
| RAG повторяет Mem0 | Разные источники — Mem0 факты, RAG файлы, Анализатор связи |
