Создано 11.07.2026. Результаты прямых тестов 8005-роутера через Python requests.

## Методика

Каждый тест — отдельный POST на `http://127.0.0.1:8005/v1/chat/completions` с моделью `8005-Router` (авто-классификация). Таймауты: приветствие 15с, обычный 30с, сложный/длинный 60с.

## Проблема: таймаут DeepSeek Reasoner на последовательных тестах

При запуске 4 тестов подряд одним Python-скриптом — третий (Pro) упал по ReadTimeout через 20 секунд.
При запуске по одному с таймаутом 60 сек — отработал штатно.

**Вывод:** DeepSeek Reasoner может тормозить на холостом старте (cold start). При параллельной/последовательной нагрузке первых секунд не хватает. Таймаут 120 сек в конфиге (PRO_TIMEOUT) — правильный, httpx-клиент должен дожидаться. Проблема именно в последовательной загрузке нескольких тестов в одном скрипте.
