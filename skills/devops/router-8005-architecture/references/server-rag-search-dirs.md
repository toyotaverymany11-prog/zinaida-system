# Server RAG — поиск по файлам сервера

## Поисковые директории
- `/opt/zinaida/meta_agent` — роутеры, скрипты, конфиги
- `/opt/zinaida/memory` — базы знаний, слепки, аналитика
- `/opt/zinaida/shared_memory` — общие заметки, логи обновлений
- `/opt/zinaida/scripts` — диагностические скрипты

## НЕ искать
- `/opt/zinaida/inbox` — 27MB, знания по отношениям (не для универсального роутера)
- `/opt/zinaida/projects` — контент-завод
- `/root/.hermes` — гигабайты sessions/skills/backup, тормозит rg
- Любые: backup/, cache/, sessions/, node_modules/, .git/

## Два этапа поиска
1. `rg` — поиск по содержимому файлов (.md .py .yaml .yml .json .toml)
2. `find` — поиск по имени файла (если имя содержит слово из запроса)

Найденные по имени файла — в начало списка (они релевантнее).

## Типы файлов
- `.md` — документация, архитектура, гайды
- `.py` — роутеры, скрипты, конфиги
- `.yaml`, `.yml`, `.json`, `.toml` — конфиги, настройки

## Производительность
- Чистый rg по 4 папкам: 0.006 сек
- find по имени: 0.05-0.1 сек
- Суммарный timeout: 3+2 = 5 сек максимум
- При timeout — ответ без контекста (graceful degradation)
