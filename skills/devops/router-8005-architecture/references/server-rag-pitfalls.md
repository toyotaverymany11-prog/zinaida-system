# Server RAG — ПРОБЛЕМЫ И РЕШЕНИЯ (11.07.2026)

## Проблема: rg timeout на больших директориях

rg (`ripgrep`) ищет файлы по ключевым словам. Если сканировать `/opt/zinaida/` целиком — 
rg тупит из-за:
- `backup/` — сотни файлов бэкапов (особенно `backup_hermes_v016*` с гигабайтами)
- `inbox/PROJECTS/Otnoshenya/knowledge/` — 2.4МБ файлов
- `/root/.hermes/` — sessions/, skills/, гигабайты

**Решение:** сузить до ключевых папок:
```python
search_dirs = [
    "/opt/zinaida/meta_agent",     # все роутеры, скрипты
    "/opt/zinaida/memory",         # SYSTEM_SNAPSHOT, memory server
    "/opt/zinaida/shared_memory",  # updates_log, roadmap
    "/opt/zinaida/scripts",        # diagnostic scripts
]
```

**Исключаемые форматы:** только `.md`, `.py`, `.yaml`, `.yml`, `.json`, `.toml`.
`.cfg`, `.conf`, `.txt` — исключены из поиска (редко содержат полезную инфу).

## Проблема: двойной вызов DeepSeek

RAG-контекст получается ПОСЛЕ того как DeepSeek уже ответил. Приходится:
1. Вызвать DeepSeek без контекста (15 сек)
2. Получить RAG-результат (0-3 сек)
3. Вызвать DeepSeek снова с контекстом (15 сек)

**Итого:** 30+ сек вместо 15.

**Решение (TODO):** внедрять RAG-контекст прямо в первый промпт DeepSeek.
Для этого `_search_rag()` должен выполняться ДО вызова DeepSeek, а не параллельно.
Либо запускать RAG синхронно (3 сек) → потом DeepSeek с контекстом (15 сек) = 18 сек.
Выигрыш: 30 → 18 сек.

## Проблема: слова-паразиты в запросе

Пользователь пишет «блядь», «ёбаный», «нахуй» — эти слова не должны идти в rg.
Фильтр стоп-слов:
```python
w not in ("блядь", "блять", "это", "что", "как", "для", "про",
          "тут", "там", "ещё", "уже", "даже", "тоже", "всё", "все",
          "нет", "давай", "сделай", "посмотри", "очень", "нужно", "можно")
```

## Проблема: Mistral analyzer может упасть

analyzer делает API вызов к Mistral. Если Mistral не отвечает (timeout/429) — 
просто идём без анализатора. Таймаут 4 сек — не блокирует ответ.
Температура 0.1 — минимум фантазий.

## Проблема: дублирование с Mem0

Mem0 хранит факты («DeepSeek ключ = sk-xxx»).
Server RAG тащит доку («вот код этого роутера»).
Это РАЗНЫЕ вещи — не дублируются.
Но если Mem0 уже знает факт — RAG может найти тот же файл.
Не критично — контекст просто содержит лишнюю инфу.
