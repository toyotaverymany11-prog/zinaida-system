# Протокол полного удаления (без следов)
## Добавлено 13.07.2026 — Указание Олега (Техник 13)

### Принцип
**Если что-то удаляется — оно должно исчезнуть полностью.**
Никаких заглушек, стubs, "занимателей имени", placeholder-контейнеров.
Удаление = 0 упоминаний в системе.

### Когда применяется
- Удаление контейнера Docker
- Удаление любого сервиса
- Удаление файла/папки
- Деактивация компонента

### Запрещено
- Создавать alpine-заглушки "чтобы daemon не восстановил"
- Оставлять пустые контейнеры/образы
- Маскировать сервисы вместо удаления

### Правильно
```bash
# Docker контейнер
docker rm -f <name>
docker rmi <image>
docker system prune -f

# Systemd сервис
systemctl stop <name>
systemctl disable <name>
rm -f /etc/systemd/system/<name>.service
find /etc/systemd/system -lname '*/<name>.service' -delete
systemctl daemon-reload

# Проверка отсутствия
docker ps -a | grep <name> || echo "нет"
systemctl list-units --all | grep <name> || echo "нет"
```

### Почему это важно
Олег: «Смысл как-то отображается там каким-то образом где-то или что блядь ты его полностью не можешь прям грохнуть чтоб о нём упоминаний не было или что блядь»

Любой след = Олег видит его в системе = гнев. Если компонент не нужен — он должен быть удалён на всех уровнях: имя, образ, конфиг, упоминания в файлах.
