Источник: глубокое исследование через 4 агента, 5 источников.

## Ключевые выводы

1. **Mem0 — де-факто стандарт** памяти для AI-агентов в 2026 (41K ⭐ GitHub, 14M загрузок, Y Combinator)
2. **Производительность:** +29.6 баллов в LoCoMo, +23.1 в LongMemEval, снижение токенов 90%, задержки 91%
3. **Точность:** +26% vs другие решения, +20-30% с гибридным поиском + zerank-1

## Entity-Scoped Memory (критично для production)

Разграничивать память по сущностям:
- `user_id` — долгосрочные предпочтения (Зинаида)
- `agent_id` — специфичные задачи (Григорий, разные агенты)
- `app_id` — изоляция между продуктами
- `run_id` — временные сессии

## Гибридный поиск с reranking

- Рекомендуется zerank-1 от ZeroEntropy (+20-30% точности)
- Модели эмбеддингов: Qwen 600M или лучше (наша e5-base 768d — хороший baseline)
- Наш threshold=0.3 даёт scores 0.78-0.84 — хорошая релевантность

## Разделение временной и постоянной памяти

- **Временная** — текущая сессия (`run_id`)
- **Постоянная** — долгосрочные факты (`user_id`)
- Не хранить логи ошибок, дампы, транзитные состояния

## Типичные ошибки пользователей

1. **Перегрузка памяти** — хранение всего подряд (97.8% шума без junk filter). Решение: наш JUNK_FILTER с инструкциями DeepSeek.
2. **Нет ретроспективного удаления** — устаревшие данные не чистятся. Решение: периодическая очистка через get_all + delete.
3. **Только cosine search** — нет гибридного поиска. Решение: zerank-1 для reranking.
4. **Нет интеграции с Redis/Valkey** — теряем скорость. Решение: наш config с кэшем.
5. **HuggingFace без токена** — rate limit на загрузку модели. Решение: HF_TOKEN для ускорения.

## Рекомендации по улучшению (очередь)

1. **Авто-сохранение из диалогов** — сейчас только ручное. Нужен механизм, который сохраняет важные факты без явной команды.
2. **agent_id scoping** — разделить память для технической работы (Григорий) и контентной (Зинаида).
3. **Ретроспективное удаление** — cron для очистки устаревших записей (старше 30 дней).
4. **Hybrid search upgrade** — zerank-1 reranking, когда появится бюджет на API.
