| Аспект | Zina2 v1 (8003) | Zina2 v2 (8005) |
|--------|-----------------|-----------------|
| Провайдеры | DeepSeek только | DeepSeek + Mistral + Ollama + GigaChat |

## Важные нюансы реализации

### DeepSeek ключ — читать из .env, НЕ из os.environ
**Ошибка v1:** `os.getenv("DEEPSEEK_API_KEY")` подхватывал systemd-ключ (мёртвый).
**Фикс v2:** читать напрямую из .env файла через `open().read()`.

```python
# ПРАВИЛЬНО (v2)
DEEPSEEK_KEY = ""
for p in [ENV_PATH, META_ENV_PATH]:
    if p.exists():
        with open(p, "r") as f:
            for line in f:
                if line.startswith("DEEPSEEK_API_KEY") and "=" in line:
                    val = line.split("=", 1)[1].strip()
                    if val and val != "***" and len(val) > 10:
                        DEEPSEEK_KEY = val
                        break
```

### Классификатор — вызывать для Zina2-Router, не форсить Flash
**Ошибка:** когда `model="Zina2-Router"`, код шёл в ветку `else: model_key = "flash"`.
**Фикс:** для неизвестных моделей вызывать `_classify_request()`, а flash форсить только при явном `"flash" in req`.

### GigaChat — ТОЛЬКО urllib, не httpx
`httpx.AsyncClient(verify=False)` с GigaChat даёт 429.
Рабочий способ: `urllib.request` + `ssl._create_unverified_context()` + `time.sleep(3)`.
Подробности: навык `devops/gigachat-integration`.

### Про триггеры Pro
- Слова-триггеры длиной < 4 символов (api, js, css) НЕ вызывают Pro
- Pro по длине — только > 300 символов (в v1 было > 150)
- Pro по коду — латиница > 40% + символы `{};:=[]`
