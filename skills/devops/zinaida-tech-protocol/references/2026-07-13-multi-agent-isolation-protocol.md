### Правило изоляции
| Что | Можно | Нельзя |
|-----|-------|--------|
| Конфиг другого профиля | ✅ `/root/.hermes/profiles/<имя>/config.yaml` | ❌ `/root/.hermes/config.yaml` (корневой) |
| MCP API | ✅ `mcp_..._use_provider_add(profile=<имя>)` | ❌ Не передавать свой profile |
| Навыки другого профиля | ✅ `/root/.hermes/profiles/<имя>/skills/` | ❌ Не трогать свои навыки |
| Systemd сервисы | ✅ Только общие (роутеры) | ❌ Не рестартить gateway без нужды |
| Файлы /opt/zinaida | ✅ Читать можно | ❌ Менять production-файлы без протокола |

### Порядок действий при починке другого агента
1. **Диагностика** — прочитать конфиг профиля, понять что сломано
2. **Не трогать свой профиль** — ни в коем случае не редактировать `/root/.hermes/config.yaml`
3. **Чинить через MCP API** — провайдеры добавлять через `mcp_hermes_studio_use_provider_add(profile=имя)`, а не редактированием конфига
4. **Gateway рестарт** — только после всех изменений, сразу проверить что сам жив
5. **Проверка** — убедиться что целевой профиль видит провайдеры и отвечает

### Добавление провайдеров в другой профиль
НЕ через редактирование custom_providers в YAML — это ломает форматирование и профиль падает.
ЧЕРЕЗ MCP API:
```
mcp_hermes_studio_use_hermes_studio_use_provider_add(
    profile="имя_профиля",
    name="имя_провайдера",
    base_url="http://127.0.0.1:XXXX/v1",
    api_key="***",
    model="имя_модели"
)
```
После добавления всех провайдеров — `systemctl restart hermes-gateway` для применения.
