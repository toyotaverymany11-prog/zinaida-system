# Урок: Потеря BrightData при чистке памяти

## Дата
13.07.2026

## Ситуация
- BrightData был установлен как основной поисковый сервис (SERP API, 5000 кредитов/мес)
- Информация о нём была **только** в MEMORY.md (одна запись)
- При плановой чистке MEMORY.md (переполнение 7465/5000) запись была удалена
- Через час Олег спросил про поиск в интернете — я начала чинить мёртвый Google API и Tavily
- Олег: «блядь ну вот этот сервис работает и ключи есть и все работает ты разве не видишь этого»

## Корень проблемы
1. **Единственная точка хранения** — BrightData был только в MEMORY.md
2. **Чистка без проверки** — удалила запись, не проверив дублирование в других местах
3. **Не загрузила навык** — `brightdata-search` существовал как навык, но я его не активировала

## Фикс (внедрён)
1. Создан навык `brightdata-search` с разделом «ПИТФОЛЛ: забыл про BrightData»
2. BrightData добавлен в `zinaida-tech-protocol` как основной поиск
3. Создан протокол `system-guarantee-protocol` — 13 точек внедрения
4. Информация записана в: MEMORY.md, Mem0, fact_store, SOUL.md, AGENTS.md, updates_log

## Правило на будущее
Любой критический сервис/инструмент — минимум 3 точки хранения:
1. MEMORY.md (кратко)
2. Соответствующий навык (подробно)
3. SOUL.md или AGENTS.md (триггер/правило)
4. Mem0 (полная копия без лимита)
