# Quality Assurance Pipeline — усиление качества через бесплатные нейросети

## Архитектура

```
DeepSeek V4 (роутер 8003, платный) — генерирует ответ
  │
  ├─→ Mistral (бесплатно) — верификация: галлюцинации, логика, стиль
  ├─→ Ollama (бесплатно) — RAG-поиск: дополнительные факты из базы
  ├─→ GigaChat (бесплатно) — редактура: русский язык, AI-маркеры
  │
  └─→ DeepSeek (опционально) — арбитр: собирает все правки в финал
```

**Ключевая идея:** бесплатные модели не ЗАМЕНЯЮТ платную, а ДОПОЛНЯЮТ. 
Они делают работу, которую платная модель сама не сделает — проверку себя со стороны.

## Использование

```bash
# Полная проверка
python3 /opt/zinaida/quality_assurance/qa_orchestrator.py --text "текст" --context "контекст"

# Только верификация (Mistral) — быстро
python3 qa_orchestrator.py --text "текст" --mode verify

# Только RAG (Ollama)
python3 qa_orchestrator.py --text "текст" --mode enhance

# Только язык (GigaChat)
python3 qa_orchestrator.py --text "текст" --mode lang

# Из файла
python3 qa_orchestrator.py --file input.txt --context "контекст задачи"
```

## Что проверяет каждый модуль

| Модуль | Модель | Проверяет | Среднее время |
|--------|--------|-----------|--------------|
| Verifier | Mistral (5 ключей) | Галлюцинации, логические дыры, стиль | ~3-5 сек |
| RAG | Ollama (3 ключа) | Поиск фактов в smm_rag.db (3975 записей) | ~0.5 сек |
| Language | GigaChat | Русский язык, AI-маркеры, штампы | ~3-5 сек |

## Режимы работы

1. **quality** (по умолчанию) — все 3 проверки параллельно
2. **verify** — только Mistral (быстро, ~5 сек)
3. **enhance** — только RAG-поиск
4. **lang** — только GigaChat

## Известные проблемы и решения

| Проблема | Решение |
|----------|---------|
| Mistral 429 (rate limit) | Автоматическая ротация 5 ключей с паузой 3 сек |
| RAG не нашёл данные | LIKE-поиск вместо FTS5 MATCH, больше ключевых слов |
| GigaChat токен протух | Авто-обновление через OAuth2 (кеш 30 мин) |
| Задержка ~10 сек | Параллельные запросы через ThreadPoolExecutor |
| Противоречивые рекомендации | DeepSeek как арбитр (опционально) |
| Mistral/Ollama/GigaChat упали | Graceful degradation — пропускаем модуль |

## Как усилить ещё

1. **Арбитр DeepSeek** — финальная сборка: DeepSeek получает отчёты от всех верификаторов и решает что применить
2. **DuckDuckGo поиск** — если RAG пуст, искать в вебе через DDGS
3. **Два раунда** — первый раунд верификации, второй — проверка что исправления не сломали стиль
4. **Group Chat агент** — добавить верификатора как отдельного агента в комнату
5. **Статистика** — логировать сколько раз каждый модуль нашёл проблему
