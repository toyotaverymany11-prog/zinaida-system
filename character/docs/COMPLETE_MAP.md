## 1️⃣ ЧТО ТАКОЕ ЗИНАИДА (системовью)
**Файл:** `fyodor/01_system_overview/system_overview.md`

Зинаида — AI-компаньон для девушек. Не бот, не психолог. **Умная подруга с памятью и структурными инструментами.**

### Что уже есть:
- ✅ **Чат с характером** — теплее ассистента, ближе к подруге, без драмы и диагнозов
- ✅ **Память** — факты, эпизоды, люди, паттерны
- ✅ **Лаборатория отношений** — 6 инструментов (главный платный модуль)
- ✅ **RAG / база знаний** — beta, но работает
- ✅ **Telegram** — быстрый канал
- ✅ **Сайт** — в разработке, будущая витрина
- ✅ **Оплата** — ЮKassa, тарифы, токены, промокоды
- ✅ **Голос и видео** — premium-слой

### Что beta:
- Тональность (иногда длинно/терапевтично)
- RAG (не продавать как экспертный мозг)
- Сайт (нужен UX-проход)
- Нагрузка (перед массовой рекламой — staging)
- Юридические документы

## 3️⃣ ТЕХНИЧЕСКАЯ СТРУКТУРА (16 блоков, 11 шагов)
**Файл:** `fyodor/03_technical_snapshot/technical_snapshot.md`

### 11 шагов обработки сообщения:
```
Пользователь → Telegram/сайт 
→ профиль пользователя 
→ проверка токенов 
→ анализатор (тип сообщения) 
→ память (факты, эпизоды, люди) 
→ RAG (если нужно) 
→ prompt layer (сборка промпта) 
→ worker/LLM (генерация) 
→ постобработка (тон, safety, кнопки) 
→ сохранение истории 
→ ответ пользователю
```

### 16 технических блоков (карточки):

| № | Блок | Роль | Ограничение |
|---|------|------|-------------|
| 1 | Telegram Bot | Быстрый вход | Слабее сайта по UX |
| 2 | Web-сайт | Главная витрина | Требует polish |
| 3 | Авторизация/профиль | Связка каналов | Дубли аккаунтов |
| 4 | Postgres | Основная БД | Резервные копии |
| 5 | Redis | Очереди и лимиты | Не замена БД |
| 6 | Worker | Тяжёлые генерации | Задержки при перегрузке |
| 7 | LLM-провайдер | Генерация ответов | **Главное узкое место** |
| 8 | Prompt layer | Характер Зинаиды | Перегрузка правил |
| 9 | Memory layer | Факты/эпизоды/люди | Управляемость |
| 10 | RAG/Knowledge | База знаний | Beta, не эксперт |
| 11 | Relationship Lab | Сценарии Лаборатории | Не диагноз |
| 12 | Billing/токены | Монетизация | Прозрачность |
| 13 | ЮKassa | Приём платежей | Webhook/бухгалтерия |
| 14 | Промокоды | Скидки/триалы | Злоупотребления |
| 15 | Voice/Video | Premium-слой | Дорого, медленно |
| 16 | Admin | Контроль системы | Нужна аналитика |

### Метафора (для презентаций):
- Telegram/сайт = двери
- Авторизация = паспорт
- Billing = касса
- Память = личная история
- RAG = библиотека
- Prompt layer = режиссёр
- LLM = генератор речи
- Лаборатория = инструменты
- Worker = цех
- Postgres = архив
- Redis = очередь на кухне
- Admin = пульт управления

### Главное узкое место:
**Внешние AI-провайдеры** (лимиты, скорость, стоимость, отказы). Не база данных.

## 6️⃣ ЧТО МОЖНО/НЕЛЬЗЯ ОБЕЩАТЬ
**Файл:** `fyodor/06_claims_safety/claims_safety.md`

**Можно:** помогает разобраться, отделить факты от тревоги, помнит контекст, видит паттерны.
**Нельзя:** замена психолога, диагнозы, точное знание чужих мыслей, предсказания, гарантии, «научно доказано».

### Safety-сценарии:
Спокойно, коротко, не продавать, направить к людям/службам.
